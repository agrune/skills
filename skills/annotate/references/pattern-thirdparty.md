# 서드파티 라이브러리 커스텀 컴포넌트 어노테이션 패턴

## 판단 기준

XYFlow, DnD Kit, Recharts 등 서드파티 라이브러리를 사용하더라도, **커스텀으로 렌더링하는 컴포넌트는 일반 JSX이므로 반드시 어노테이션 대상이다.** 라이브러리가 자체적으로 렌더링하는 내부 요소만 제외한다.

| 구분 | 어노테이션 | 이유 |
|------|-----------|------|
| 커스텀 노드/셀/아이템 컴포넌트 | **O** | 일반 JSX로 렌더링되어 DOM에 data-* 속성 전달됨 |
| 라이브러리 내부 SVG (엣지, 그리드, 축 등) | X | 라이브러리가 직접 렌더링하여 속성 전달 불가 |
| 라이브러리 내장 컨트롤 (줌, 미니맵 등) | X | 줌/팬은 pointer 이벤트로 처리 가능, 어노테이션 불필요 |

## 판단 방법

1. 컴포넌트가 프로젝트 코드에서 정의되었는가? → **어노테이션 대상**
2. 라이브러리의 내장 렌더러가 그리는가? → 제외
3. 확실하지 않으면: 해당 컴포넌트의 JSX를 보라. `data-agrune-*` 속성을 DOM 요소에 직접 전달할 수 있으면 어노테이션 대상이다.
